

# System Update Plan

This plan addresses all requested updates: fan profiles with badges/points, clickable player cards for officials/fans on stats page, captain game stats, assistant coach contribution removal, weekly stat history, game type/venue tracking, league integration, and more.

## 1. Fan Profile System (Login, Badges, Points, Super Fan)

### Database Migration
- Add columns to `members`: `fan_badge TEXT DEFAULT NULL`, `fan_points INT DEFAULT 0`, `favourite_moment TEXT DEFAULT NULL`
- Fan IDs are already generated by `addPlayer` with role `fan`

### Changes
- **`addPlayer` in TeamDataContext**: When role is `fan`, also generate a fan ID like `SCF-F01` (new prefix) to distinguish from player IDs. This makes them loginable with their ID directly (no PIN needed, like players).
- **`team-data.ts`**: Update `authenticateMember` to also match `SCF-F` prefix IDs without PIN.
- **`PlayerProfile.tsx`**: When `user.role === "fan"`, hide attendance and contributions sections. Show "Favourite Team Moment" text area (saved to `members.favourite_moment` via new context method). Show fan badge and points decoratively.
- **`OfficialProfile.tsx` (Manager)**: Add "Fan Management" card â€” list all fans, buttons to assign badges (`Super Fan`, `Stadium Regular`, `Top Fan`) and add points. Each attendance/match recorded adds 2 points. On Sunday, the fan with most points auto-gets `Super Fan` badge.
- **`Stats.tsx`**: In the Fans section, show each fan's badge, points, and favourite moment (visible Mon-Fri only). When clicking a fan or official card, open a player-card-style dialog showing their profile pic, name, role, stats (if applicable), badge, and points.

## 2. Clickable Player Cards for Officials & Fans on Stats Page

- **`Stats.tsx`**: Wrap each official/fan card in a clickable button. On click, open a `Dialog` showing the same layout as a player card: avatar, name, role, position, stats (for captains/Fadhir), badge/points (for fans).
- Add state: `selectedMemberCard` and a `MemberCardDialog` component.

## 3. Captain & Fadhir Game Stats on Their Profile

Currently `getCaptainStatCards()` exists and is shown for `isCaptain`. Extend:
- **`OfficialProfile.tsx`**: Show the captain stats card also for `isFadhir` (Fadhir has role `finance` but needs player stats). Change condition from `isCaptain` to `isCaptain || isFadhir`.
- Ensure the Manager's Team Stats Editor dropdown includes Fadhir and all captains (currently it only shows `playerMembers` which filters by `role === "player" || role === "captain"`). Add Fadhir to this list.

## 4. Remove Monthly Contributions from Assistant Coach Profile

- **`OfficialProfile.tsx`**: Change `showContributions` from `!isFabian` to `!isFabian && !isAssistantCoach`. This hides the "My Monthly Contributions" section for the assistant coach.

## 5. Weekly Stat History (Permanent Records per Player)

### Database Migration
- Create table `weekly_stats_log`: `id UUID PK, player_id TEXT NOT NULL, week_start TEXT NOT NULL, goals INT DEFAULT 0, assists INT DEFAULT 0, games_played INT DEFAULT 0, saves INT DEFAULT 0, clean_sheets INT DEFAULT 0, aerial_duels INT DEFAULT 0, tackles INT DEFAULT 0, interceptions INT DEFAULT 0, blocks INT DEFAULT 0, clearances INT DEFAULT 0, successful_tackles INT DEFAULT 0, direct_targets INT DEFAULT 0, direct_shots INT DEFAULT 0, created_at TIMESTAMPTZ DEFAULT now()`
- Add RLS: public read/insert/update/delete (matching existing pattern)

### Logic
- **`TeamDataContext.tsx`**: When `updatePlayerStats` is called, also compute the delta (new - old values) and insert/upsert into `weekly_stats_log` for current week. Add `loadWeeklyStatsLog(playerId)` method.
- **`PlayerProfile.tsx`**: Add "Weekly Activity Log" section showing stats per week in a neat accordion/list, sorted newest first. Each entry shows: week date, goals scored that week, assists, tackles, etc. Only show non-zero stats.
- **Export**: The "Export Profile" button (Fri-Sun) generates the full detailed DOCX with all weekly records. Add a second "Export This Week" button for just the current week's data.
- **Image fix**: The profile pic URL uses a cache-buster `?t=...` which may break the `fetchImageAsBuffer` in docx-export. Fix by stripping query params before fetching, or handle the redirect properly.

## 6. Game Type, Venue, and Date on Results

### Database Migration
- Add columns to `game_scores`: `game_type TEXT DEFAULT 'friendly'`, `venue TEXT DEFAULT NULL`

### Changes
- **`GameScore` interface**: Add `gameType` and `venue` fields.
- **`TeamDataContext.tsx`**: Update `addGameScore` and `loadGameScores` to include new fields. Update `updateGameScore` to handle them.
- **`OfficialProfile.tsx` (Add Recent Results)**: Add dropdowns for game type (`Friendly`, `League`, `Amateur`) and venue (`Kanjuri Grounds`, `G-Town`, `School Field`). Pass to `addGameScore`.
- **`Results.tsx`**: Show game type as a bold label badge (e.g., "LEAGUE", "FRIENDLY", "AMATEUR") and venue on each result card.
- **`Stats.tsx`**: Show game type and venue in match report cards.

## 7. League/Amateur Integration with Add Score

When manager adds a result and specifies `League` or `Amateur`:
- After inserting the game score, auto-find Suncity's row in `league_teams` (where `is_own_team = true` and matching division) and update: increment `played`, set `won/drawn/lost` based on score, update `goal_difference` and recalculate `points`.
- This happens in `addGameScore` in `TeamDataContext.tsx`.

## 8. Amateur Standings on Everyone's Stats Page

- **`Stats.tsx`**: Already has league standings loading but doesn't display them for everyone. Add League Standings and Amateur Standings tables (read-only) on the Stats page, similar to Results page display. Load `league_teams` from Supabase.
